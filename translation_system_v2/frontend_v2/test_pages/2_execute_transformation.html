⚡ 阶段2: 执行转换

架构阶段: SPLIT_COMPLETE → EXECUTING → COMPLETED

输入: Session ID OR Excel文件（数据状态N + 任务表）

输出: Session ID（数据状态N+1） + 可导出验证

💡 架构原则: 任务拆分表 → [转换器] → 数据状态N+1
• 输入灵活性：继续已拆分Session OR 上传文件并拆分再转换
• 转换器根据任务表中的指令执行转换
• 输出可验证：导出Excel查看转换后的中间状态

📥 选择输入方式

🔗 使用Session ID
📁 上传文件并执行
💡 从 "阶段1: 任务拆分" 获取已完成拆分的Session ID
必须是 stage=SPLIT_COMPLETE 的Session
📝 上传文件将自动拆分任务并执行转换（快速模式）
💡 Processor自动选择逻辑:
• rule_set=translation → processor=llm_qwen（LLM翻译）
• rule_set=caps_only → processor=uppercase（大写转换）

建议: 使用自动选择
同时处理的任务批次数，建议5-10
💡 启用后，翻译时会根据术语表进行术语匹配和提示
支持上传JSON格式术语表或选择已有术语表
支持两种格式：
1. 简化格式: {"术语": "translation"}
2. 标准格式: {"id": "xxx", "terms": [{...}]}
