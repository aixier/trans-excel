🎯 阶段1: 任务拆分

架构阶段: CREATED → INPUT_LOADED → SPLIT_COMPLETE

输入: Excel文件 OR Session ID（数据状态N）

输出: Session ID + 任务拆分表 + 可导出验证

💡 架构原则: 数据状态N → [拆分器] → 任务拆分表
• 输入灵活性：上传新文件 OR 使用已有Session的数据
• 拆分器根据规则生成任务表（描述如何转换）
• 输出可验证：导出任务表Excel查看拆分结果

📥 选择输入方式

📁 上传Excel文件
🔗 使用已有Session
📝 上传新文件将创建新Session，文件内容作为 数据状态0
💡 从已有Session继承数据，可以用不同规则重新拆分
使用场景：
• 阶段2完成后，想用不同规则拆分（如CAPS）
• 验证某个数据状态的不同拆分策略
支持: CH,EN,TR,TH,PT,VN,IND,ES,AR,FR,DE,JP,KR,RU,TW,BR,IT,PL,NL,MS,HI
💡 规则集决定任务类型:
• translation: 空单元格、黄色重翻、蓝色缩短、普通翻译
• caps_only: 仅拆分CAPS表格的大写转换任务
